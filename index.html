My journey!

  • Block coding!

    I started my journey with a coding method called block coding. Block coding is a method used in computer science and information theory to represent and transmit data in a structured and organized manner. It involves dividing data into fixed-size blocks or groups of bits, each containing a predefined number of binary digits (0s and 1s). These blocks are created with specific patterns and redundancy information, which helps in error detection and correction during data transmission or storage. So instead of typing in every single code, you can use blocks and drag and drop!

  • Text based programming language

    After block coding, I needed to go to the next level of coding which is text based programming. Text-based programming is a method of writing and creating computer programs using textual instructions and commands in a programming language. In this approach, programmers use human-readable text to communicate with the computer and provide it with a set of instructions to perform specific tasks or calculations.Text-based programming relies on programming languages, each with its own syntax and rules. Examples of popular text-based programming languages include Python, Java, C++, JavaScript, and many others. Programmers write code in these languages to create software applications, websites, and other computational solutions.

    Basic Structure of Coding!

    Variables!

    A variable is a temporary container to store information, it is a named location in computer memory where varying data, like numbers and characters, can be stored and manipulated during the execution of the program.

    Conditional Statements!

    A type of coding instruction used to compare values and express and make decisions. A conditional statement tells a program to execute an action depending on whether a condition is true or false. It is often represented as an if-then or if-then-else statement.

    Functions!

    Functions are "self contained" modules of code that accomplish a specific task. Functions usually "take in" data, process it, and "return" a result. Once a function is written, it can be used over and over again. Functions can be "called" from the inside of other functions.
